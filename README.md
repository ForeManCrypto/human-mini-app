# Proof of Human — Telegram Mini App

A Telegram bot + mini app that gates entry to private groups using [ShareRing Me](https://sharering.network) biometric identity verification. Only real, verified humans get approved — no bots, no fakes.

## How it works

```
User requests to join group
        │
        ▼
  Bot sends welcome DM  ──→  User taps "Verify Now"
        │                           │
        ▼                           ▼
   Mini App opens           ShareRing Me app
  (shows QR code)        scans QR & approves
        │                           │
        ▼                           ▼
  Polls Worker ◄──── ShareRing POSTs /verified
        │
        ▼
  Telegram join request approved ✅
```

1. A user requests to join a private Telegram group
2. The bot sends them a DM with a guided flow
3. The user taps **Verify Now** — a Telegram Mini App opens
4. A QR code is generated by the ShareRing SDK
5. The user scans it with **ShareRing Me** on their phone
6. ShareRing calls back the Cloudflare Worker webhook
7. The Worker approves the Telegram join request via the Bot API
8. The user is welcomed into the group

## Prerequisites

- A **Telegram Bot** token from [@BotFather](https://t.me/BotFather)
- A **ShareRing** account with a query configured at [link.sharering.network](https://link.sharering.network)
- A **Cloudflare** account (free tier is fine) with Workers and KV enabled
- A **Railway** account (or any host that can run Python)

## Deployment

### 1. Frontend — GitHub Pages

Fork this repo, enable GitHub Pages on the `main` branch (root `/`).
Your mini app URL will be: `https://<your-username>.github.io/human-mini-app`

Update `MINI_APP_URL` in `bot/bot.py` and `WORKER_URL` in `index.html` to match your deployment.

### 2. Cloudflare Worker

```bash
cd worker
npm install -g wrangler
wrangler login
wrangler deploy
```

Set the required secrets:

```bash
wrangler secret put BOT_TOKEN
wrangler secret put WORKER_SECRET        # random string shared with the bot
wrangler secret put SHARERING_WEBHOOK_SECRET  # from ShareRing dashboard
```

Create a KV namespace and update the `id` in `wrangler.toml`:

```bash
wrangler kv namespace create SESSIONS
```

Configure your ShareRing webhook URL to point to:
`https://<your-worker>.workers.dev/verified?secret=<SHARERING_WEBHOOK_SECRET>`

### 3. Telegram Bot — Railway

```bash
cd bot
# Push to a Railway project linked to this repo
```

Set environment variables in Railway:

| Variable | Description |
|----------|-------------|
| `BOT_TOKEN` | Telegram bot token |
| `MINI_APP_URL` | GitHub Pages URL of the frontend |
| `WORKER_URL` | Cloudflare Worker URL |
| `WORKER_SECRET` | Shared secret (same as wrangler secret) |
| `WEBHOOK_HOST` | Railway public domain (set automatically as `RAILWAY_PUBLIC_DOMAIN`) |
| `PORT` | Port to listen on (default: `8080`) |

### 4. Configure the Bot

In **BotFather**:
- Set the bot's **Menu Button** / Mini App to your GitHub Pages URL
- Enable **Allow Groups** and **Group Privacy** off so it can see join requests

In your **Telegram group**:
- Add the bot as an **administrator**
- Enable **Approve New Members** (join requests)

## Security

The worker implements several security controls:

- **C1** — ShareRing webhook verified via shared secret (query param)
- **C2** — `/session` endpoint accepts calls from the frontend (bot-side pre-registration is scaffolded for future hardening)
- **C3** — CORS restricted to `foremancrypto.github.io`, `web.telegram.org`, `telegram.org`
- **C4** — Telegram webhook secret derived from `BOT_TOKEN` to prevent fake update injection
- **H2** — Sessions deleted from KV after successful verification
- **M5** — Per-IP and per-user rate limiting on all endpoints
- **M6** — Input validation (UUID format, numeric user/chat IDs)

## Local Development

### Bot

```bash
cd bot
pip install -r requirements.txt
BOT_TOKEN=xxx WORKER_URL=https://... python3 bot.py
```

> Note: The bot runs in webhook mode. For local dev, use [ngrok](https://ngrok.com) or switch to polling mode by replacing `app.run_webhook(...)` with `app.run_polling()`.

### Worker

```bash
cd worker
wrangler dev
```
